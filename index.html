01 Scope

Invariant Labs builds infrastructure where correctness is the product. We develop deterministic data processing libraries, standards-grade conformance tooling, and engineering practices that produce governed evidence of correctness — not assertions of it.

The stack operates at the lowest layers of the software supply chain: IEEE 754 floating-point formatting, JSON canonicalization, and deterministic build verification. These are the layers where one silent mismatch invalidates every assurance built above them.

The stack is designed for environments where ambiguity is a defect, leniency is a liability, and rejection is the safe outcome. Unit tests provide confidence. Evidence bundles provide proof. We do not ship on confidence alone.

02 Conformance Evidence
Evidence methodology — core differentiator

Releases are gated on offline replay evidence — not test passage alone. A replay harness executes canonical operations across distributions and architectures, captures four independent SHA-256 digest streams, and requires byte-identical output across the full matrix before any release is permitted.

Method Offline replay · 12 nodes · 5 cold replays · 60 independent executions
Nodes Debian · Ubuntu · Alpine · Fedora · Rocky · openSUSE — container + VM
Streams Canonical digest · Verify digest · Failure class digest · Exit code digest
Binding SHA-256 chain: evidence → bundle → vectors → source commit
Gate Hard release gate · evidence-required · no default path

The evidence does not claim all possible builds are deterministic. It claims this exact binary, built from this exact source commit, produced identical output across an explicit replay matrix. The boundary between engineering evidence and overclaiming is the boundary we govern.

03 Infrastructure
Current baseline — available now
json-canon
Go · Apache 2.0 · github.com/lattice-substrate/json-canon
Deterministic RFC 8785 JSON Canonicalization Scheme. Strict RFC 8259 parser rejecting lone surrogates, duplicate keys, and noncharacters. ECMA-262 number formatting with even-digit tie-breaking. UTF-16 code-unit key sorting. Thirteen-class failure taxonomy with stable exit codes. Seven independent resource bounds. Zero dependencies. 286,362 cross-engine oracle test vectors.
jcs-schubfach
Go · Apache 2.0 · github.com/lattice-substrate/jcs-schubfach
Fixed-width Schubfach digit generation for RFC 8785 number formatting. Same ECMA-262 conformance contract, same output bytes, no multiprecision arithmetic. 31.3% geometric mean throughput improvement over Burger-Dybvig with zero conformance regressions.
Lattice Substrate
GitHub · github.com/lattice-substrate · 11 repositories
Organization home for deterministic infrastructure primitives. Four digit-generation algorithms across two languages (Go, Rust), benchmark laboratory, conformance harness, and research artifacts.
